*"3개월 개발기" 2편. 3명이 어떻게 Claude Code·Gemini·Cursor 를
역할별로 나눠서 썼는지, 어디서 실수했는지, 그리고 3.5개월 동안 무엇이
관행으로 굳었는지.*

## 초기에 우리가 한 실수

Ep 1 에서 다룬 전제 조건 — 3명, RTX 4070 한 대, 개인 시간 — 이
그대로라면 AI 도구가 *구원* 처럼 보인다. "내가 3배로 코딩할 수 있으면
3인 팀이 9인 팀" 같은 환상이 쉬웠다.

첫 2주 동안 우리는 Claude Code 를 *범용 엔지니어* 처럼 썼다. 태스크
하나 던지고 "이거 구현해줘" 하고 결과를 받아서 머지. 그럴듯한 코드가
나왔다 — 몇 시간 뒤 코드를 다시 읽으면 우리 아키텍처와 미묘하게 다른
가정을 깔고 있었다. 피처 그룹 이름을 조금 다르게 쓴다거나, config
경로를 임의로 만들어낸다거나, 관심사 분리 규칙을 무시하고 train.py
에 전처리 로직을 섞어넣는 식.

이걸 잡으려고 추가 시간이 들었고, 잡고 나면 그 시간 절약 효과가
거의 사라졌다. "AI 가 3배 빠르게 코딩한다" 는 감각은 진짜가 아니었고,
"AI 가 코드를 쓰고 우리가 그걸 교정하는 데 드는 순시간" 이 실제 진행
속도였다. 그리고 교정 시간이 작성 시간의 70% 를 넘기 시작하면 순가치는
마이너스로 떨어진다.

이 실수를 인정하고 나서 질문이 바뀌었다. "AI 로 무엇을 하게 할까?"
가 아니라 "AI 의 어떤 역량을 어떤 단계에 두어야 우리가 진짜 빨라지는가?"

## 3단계 재구성

3.5개월에 걸쳐 우리가 안정화시킨 분업은 이것이다.

**1단계 — 설계 대화 (Claude 웹 인터페이스).**
파일에 쓰지 않는 긴 대화. 아이디어를 던지고 트레이드오프를 탐색하고
숨은 가정을 드러낸다. "이 전문가를 하이퍼볼릭 기하 기반으로 하면 어떤
장점이 있고 어떤 비용이 드나?", "Black-Litterman 앙상블은 왜 기각
대상인가?" 같은 질문. 이 단계의 산출물은 *결정과 그 근거* 이고, 이는
손으로 정리한 설계 문서 또는 마크다운으로 옮겨진다.

이 단계에서 코드는 쓰지 않는다. 코드를 쓰면 대화가 "이 코드 맞지?"
검토 모드로 전환되면서 트레이드오프 탐색이 멈춘다.

**2단계 — 구현 (Claude Code, 터미널).**
설계가 확정되고 나서 Claude Code 에 *명시적 컨텍스트* 와 함께 던진다
— 어느 파일, 어느 인터페이스 계약, 어느 기존 패턴을 따를지. 작은
단위로 자르고, 한 세션은 한 작업만. 에이전트가 diff 를 제안하면
사람이 읽고 거부 또는 승인.

이 단계에서 가장 중요한 규칙 — "Claude Code 가 쓴 코드를 읽지 않고
커밋하지 않는다." 단순한 규칙이지만 초반에는 피곤해서 지키기 어렵다.
지키지 않으면 위 "교정 시간 70%" 로 돌아간다.

**3단계 — 리뷰와 디버깅 (Gemini + 사람).**
버그가 났거나 설계 선택이 의심스러울 때, Gemini 의 대용량 컨텍스트
읽기로 2차 의견을 받는다. 증상, 관련 코드, 우리 가설을 한꺼번에
붙여서 "다른 해석 가능한 루트 원인은 뭐가 있나?" 를 묻는 방식. Gemini
는 코드를 *쓰는* 역할이 아니라 *읽는* 역할이다. 쓰는 일은 Claude
Code 로 단일화되어 있어야 컨벤션이 흐트러지지 않는다.

Cursor 는 엔지니어 2명이 자동완성 흐름이 필요할 때만 썼다. 아키텍처
결정이나 대규모 편집에는 사용하지 않았다.

## 구독료의 함정

처음엔 3명 모두 Claude Pro (\$20/월) 를 구독했다. 한 달 뒤 사용량을
봤더니 PM 은 항상 한도를 쳤고 엔지니어 2명은 30-40% 정도만 쓰고
있었다. 집단적으로 \$60/월을 쓰지만 PM 은 용량 부족이고 엔지니어는
남는 용량이 놀고 있었다.

재조정: PM 은 Claude Max (\$100/월) 로 승격, 엔지니어 2명은 Pro 유지.
추가로 Gemini 2.5 Pro (\$20/월) 를 PM 이 Gemini 역할 전용으로 구독,
Cursor (\$20/월) 는 엔지니어 공용 계정.

최종 비용 구조 (개인 지갑에서):
- Claude Max (PM) — \$100/월
- Claude Pro × 2 (엔지니어) — \$40/월
- Gemini Pro — \$20/월
- Cursor — \$20/월
- AWS 스팟 + S3 — 월 평균 \$60-80

합계: 월 약 \$240-260, 3.5개월 ≈ \$900. 엔지니어 1명의 한 주치 임금
정도. "AI 도구가 비싸다" 는 느낌은 subscription 수준이 아니라
*잘못된 단계에 잘못된 도구를 쓸 때의 시간 낭비* 수준에서 온다는
점이 분명해졌다.

## 컨텍스트 위생

3.5개월 동안 가장 값비싼 교훈 하나는 "세션 수명" 이다. Claude Code
세션을 며칠씩 이어 쓰면, 컨텍스트가 이전 작업의 잔해로 혼탁해진다.
이전에 논의한 가설, 폐기한 접근, 수정 전 코드 — 이 모든 것이 현재
태스크 판단에 간섭한다.

정착한 규칙:
- 한 태스크 끝 = 세션 종료 (다음 태스크는 새 세션)
- 한 세션 안에서 주제가 바뀌면 `/compact` 강제
- 영속적 규칙은 세션 이력이 아니라 `CLAUDE.md` 에 (프로젝트 루트)
- 대용량 참조 자료 (논문, 온프렘 문서) 는 세션 첫 메시지에 한 번만,
  이후엔 referenced 방식으로

`CLAUDE.md` 가 특히 중요했다. 우리 프로젝트의 `CLAUDE.md` 는 지금
6개 섹션, 하드코딩 금지, 관심사 분리, 비용 관리, 오케스트레이션
효율성 같은 규칙을 담고 있다. 세션마다 이걸 다시 설명하는 건 불가능
하다. 한 번 써두고 에이전트가 매번 자동 참조하게 하는 것이 유일한
확장 경로였다.

## 3.5개월 동안 굳은 패턴

**Plan-first.** 새 기능은 반드시 plan 문서부터. Claude Code 가 바로
구현에 뛰어들지 못하게 막는 장치. 한 번 시작하면 5분 안에 "plan 없이
시작했네" 하고 되돌리는 버릇이 생겼다.

**Interface-contract 검증.** 병렬로 여러 에이전트 작업을 돌린 뒤에는
*반드시* 인터페이스 계약 검증 단계를 별도로 돌린다 — "A 가 저장하는
키 이름" 과 "B 가 읽는 키 이름" 이 일치하는지. 에이전트들은 각자의
맥락에서 그럴듯한 이름을 쓰는데, 두 에이전트의 "그럴듯한 이름" 이
서로 다르면 런타임까지 가서야 실패한다.

**로컬 먼저, SageMaker 는 마지막.** SageMaker 에서 코드 디버깅을 하면
Job 제출당 \$0.50+ 비용이 즉시 발생한다. 로컬 GPU 에서 1 epoch
end-to-end 성공 → 그다음 SageMaker. 이걸 지키지 않으면 한 실수당
Job 10번 = \$5 가 그냥 증발한다.

**완료 보고 기준.** "완료" 라고 보고하려면 네 단계 검수를 통과해야
한다 — 컴파일 검증, 인터페이스 계약 검증, 하드코딩 스캔, 관심사
분리 검증. 일부만 한 경우는 "일부 OK, 나머지 미검증" 으로 명시적으로
부분 보고. 이 습관이 없었으면 "완료" 와 "실제 작동" 의 간극이
누적되었을 것이다.
